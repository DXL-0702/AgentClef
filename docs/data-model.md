# Data Model

## Design Principle

AgentClef stores a structured music model as the system source of truth. MIDI, MusicXML, PDF, and image outputs are export products, not internal truth.

The v0.1 model combines:

- Audio time coordinates.
- Musical beat coordinates.
- Editable note events.
- Optional chord events.
- Local uncertainty markers.
- Agent candidate edits.
- User-confirmed revisions.

## Object Graph

```text
Project
├── AudioAsset
├── TranscriptionJob
├── DraftScore
│   ├── BeatGrid
│   ├── Track
│   │   └── NoteEvent
│   ├── ChordEvent
│   └── UncertaintyMarker
├── AgentMessage
│   └── SelectionRange
├── CandidateEdit
└── Revision
```

## Persistence Baseline

v0.1 persists the core object graph in PostgreSQL through SQLAlchemy models and Alembic migrations.

The initial persistence baseline creates tables for:

- Project.
- AudioAsset.
- TranscriptionJob.
- DraftScore.
- AgentMessage.
- CandidateEdit.
- Revision.

Structured DraftScore, Agent context, CandidateEdit, and Revision payloads are stored as PostgreSQL JSONB fields. Stable resource metadata remains in typed relational columns.

## Core Objects

### Project

A transcription review workspace for one piece or audio source.

Contains:

- Project metadata.
- Linked audio assets.
- Transcription task records.
- Current DraftScore.
- Agent and revision records.

### AudioAsset

The uploaded and normalized audio source used for transcription and evidence playback.

Contains:

- Original file metadata.
- Normalized audio metadata.
- Duration in seconds as `duration_seconds`.
- Format.
- Processing status.
- Storage location.

### TranscriptionJob

An asynchronous task that turns an AudioAsset into a DraftScore.

States:

```text
created
-> uploaded
-> preprocessing
-> transcribing
-> postprocessing
-> draft_ready
```

Failure states:

```text
upload_failed
preprocessing_failed
transcription_failed
postprocessing_failed
```

### BeatGrid

The mapping between audio time and musical position.

Contains:

- Tempo data.
- Meter.
- Beat positions.
- Measure positions.
- Audio-time to beat-position mapping.
- Confidence metadata.

### DraftScore

The current editable draft score for a project.

Contains:

- BeatGrid reference.
- Track collection.
- NoteEvent collection.
- Optional ChordEvent collection.
- UncertaintyMarker collection.
- Current version.

### Track

A logical container for music events.

v0.1 tracks:

- `main_melody`
- `chords` when chord candidates are enabled

### NoteEvent

The primary editable note unit.

Contains:

- Stable identifier.
- Track reference.
- Pitch.
- Audio start and end time.
- Musical start beat.
- Musical duration.
- Source: model, user, or confirmed Agent edit.
- Confidence metadata.
- Notation hints.

### ChordEvent

A chord label or candidate set over a time range.

Contains:

- Label.
- Candidate labels.
- Audio time range.
- Beat range.
- Confidence metadata.
- Source.

### SelectionRange

The local score or audio region selected by the user.

Selection can target:

- One note.
- Multiple notes.
- A measure range.
- A time range.
- A chord.
- An uncertainty marker.

### UncertaintyMarker

A review target generated by the pipeline or Agent.

Types:

- `pitch_uncertain`
- `rhythm_uncertain`
- `duration_uncertain`
- `chord_uncertain`
- `beat_alignment_uncertain`

### AgentMessage

A conversation message bound to score context.

Contains:

- Message role.
- Message content.
- SelectionRange reference.
- DraftScore version reference.
- Related NoteEvent and ChordEvent references.
- CandidateEdit references.

### CandidateEdit

A structured edit proposal that has not yet been applied.

Supported v0.1 edit types:

- `change_pitch`
- `change_duration`
- `move_note`
- `add_note`
- `delete_note`
- `change_chord`

### Revision

A confirmed change applied to DraftScore.

Contains:

- Revision identifier.
- DraftScore version.
- Source.
- CandidateEdit reference when applicable.
- User confirmation metadata.
- Affected range.
- Summary.
- Timestamp.

## Source of Truth Rules

- DraftScore is the current score truth.
- AgentMessage is contextual history, not score truth.
- CandidateEdit is a proposal, not a mutation.
- Revision is the record of an applied mutation.
- Export files are generated from DraftScore.
